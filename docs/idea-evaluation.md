# Loom 想法评估

> 由 4 个独立视角（技术可行性 / 新颖性 / 系统架构 / Token 经济性）并行评估得出。整体结论：**方向对，但"组装整个 GitHub"不现实**——四个视角独立收敛到同一个命门：**可组合性(composability)是源作者预先投入干净接口边界的结果，无法被检索或匹配出来**。可落地形态是"受控生态内的配方式组装"（shadcn/插件模型）。

## 跨视角的共识（最重要）

1. **可组合性不可检索**：兼容性是涌现属性，取决于源 repo 是否有干净接口边界——绝大多数真实 repo 没有。embedding/BM25 只能检索表面相似，不等于行为等价或可拼接。
2. **接缝(seam)才是产品核心**：真正的难点和护城河都在"两段异源代码能否在接缝处低成本、合法拼接"。重心要从"相似度"转移到"接缝建模"。AI 的主战场不是生成部件，而是**生成把部件粘起来的 adapter/胶水层**。
3. **"省 token"论证可能被实测推翻**：复用省的是 output token，但要写对 diff，模型必须先读懂多个陌生项目（海量"理解 token"）。理解成本通常压倒生成成本。只在"匹配质量高 + 复用单元大且自包含"时净赢。
4. **多源 Frankenstein 组装是最差模式**：拆得越细、来源越杂，接缝越多，风格越乱，质量与 token 同时崩盘。整套复用单一连贯 starter 赢面大。

## 各视角要点

### 技术可行性
- **强项**：`diff + 已知良好基线`正是编码 Agent 最擅长、最可验证的工作模式；窄域先例（shadcn copy-paste 组件、cookiecutter/Yeoman/Plop、Backstage 模板）已验证"受控复用+微调"可落地。
- **致命弱点**：依赖地狱（React17 vs 19、ESM vs CJS、Maven 依赖仲裁、Python/Java 单命名空间无法多版本共存）；隐式耦合（模块抽出会拖出全局状态/DI/中间件，是"复制式复用"几十年失败的根因）；构建系统不兼容；License 法律雷区（GPL/AGPL 传染、GPLv2 与 Apache2.0 不兼容）。
- **关键洞察**：唯一可落地形态是`受控注册表 + 强制接口契约 + 单生态构建链`，本质是聪明的脚手架+组件市场，而非"组装整个 GitHub"。

### 新颖性与差异化
- **真正的新颖性**不是"复用代码"（人人都做，护城河为零），而是**范式倒置**：把默认从`生成-再修复`(Copilot/v0/bolt) 改为`检索-组装-再 diff`。
- **但倒置只有配上"可组装性层"才形成护城河**：把原始 repo 规范化为带接口类型、带兼容性评分的可组装单元。裸爬 GitHub 不可防御，会退化成"更花哨的模板市场"。
- **复利型数据护城河**：记录每次组装是否编译/测试/上线成功，用它给未来匹配排序——这类反馈难被复刻。

### 系统架构
- 四层天然分层：`语义层(想法空间) / 索引层 / 物化层 / 补丁层(diff)`，每层可独立演进。
- `diff-on-assembly` 与 Kustomize/OCI 分层、Nix overlay 同构，可审计、可重放。
- **弱点**：固定层级树（网站→电商）与现实多维度（领域×架构×技术栈×部署目标）冲突，会与语料漂移；把"想法"当一个点，但组装发生在能力/模块粒度，缺"想法→能力清单"分解层。
- **关键洞察**：谁建模了接缝，谁就掌握护城河；只做向量检索的版本会沦为又一个"相似项目搜索引擎"。

### Token 经济性与质量
- **真实节省**：样板密集输出（Web 骨架、CRUD、auth/支付）复用省掉最贵的 output token；复用代码经实战检验、bug 率低。
- **真实成本**：input token 膨胀（为一小块功能读入整个项目）；接缝是 bug 与调试 token 集中区；检索是"固定税"，匹配越差越亏。
- **止损机制**：用`适配比 = diff 行数 / 复用行数`实时判断，超阈值（如 >0.4）就转为从零生成。

## 综合给出的可执行建议（汇总）

- 范围收窄为**策展式注册表 + 强制 adapter 接口契约**，只沿预定义 seam 组装。
- 每个 core **锁定单一生态与构建链**，v1 拒绝跨语言/跨 bundler 组装。
- **License 合规闸门前置**：SPDX 标注 + 兼容矩阵 + ScanCode 扫描，默认排除 copyleft。
- **用证据代替相似度幻觉**：每个组装候选跑 golden 集成测试，"能否跑通"作为可组合性的唯一真实信号。
- **重写阈值**：相似度低于阈值或 diff 超预算时，自动回退到从零生成。
- **把 token 预算投在接缝上**：为每个跨源边界显式生成 adapter + 接缝测试。
- **度量回路**：埋点 `diff 体积 / 一次跑通率 / 含适配的实际总 token`，用数据验证组装是否真优于生成。
